Getting Started

This tutorial describes the process of creating a new CUTE project where all the code (test code and tested code) is in the same project. For more information on how to organize your Code and the different project types see the linked chapters.

Step 1: Creating a project

  1. Select File > New > C++ Project

  2. In the Project name field, type a name for the project, such as HelloCute
  3. In the Project types list, select CUTE Project. This is the most simple CUTE project type.
  4. Select a toolchain from the Toolchain list.

  5. Click Next

  6. In the CUTE Version list select the latest version of CUTE installed on your system.

    See ... to install an other version of CUTE.

  7. If your using gcc with gcov check the Enable coverage analysis using gcov checkbox.
  8. Click Finish

Step 2: Reviewing and building the project

The wizard created a C++ project with 2 source folders. The cute folder contains the CUTE headers, the src folder contains the Test.cpp with a single failing test case. If you enabled the coverage analysis your project has 3 build configurations:

Release
Build configuration for releases, without debug information.
Debug
Standard debug config.
Debug Gcov
Debug config with gcov compiler settings.

Adding a new test function

  1. Open the Test.cpp file in the editor.
  2. Add a new test function by pressing CTRL+ALT+F change the function name by typing the new name.
  3. Save the changes by clicking File > Save or pressing CTLR+S
  4. Build your project by pressing CTRK+B or select the project in the Project Explorer tab and click Project > Build Project
  5. Read through the build messages in the Console view. The project should build successfully

Step 3: Running the tests and review the results

  1. Within the C/C++ perspective click Run > Run Configurations..
  2. Select CUTE Test and press the New button

    A new run configuration is created. Its name and path are provided by the project and the build configuration.

  3. Click Run

    Now you should see the test application running in the console view. This should also open the CUTE Test Results view

    Double clicking on a failing test opens and selects the failing assert in the editor.

    Tip: to prevent the Console to hide the CUTE Test Result view on every run you can uncheck the Show Console When Standard Out Changes button in the upper right corner of the view

  4. If you enabled the coverage analysis the code will be highlighted.
